Glossary

Plain-English definitions for every acronym used across the roadmap docs. Type to filter.

Frameworks & Standards
NIST CSF
The National Institute of Standards and Technology's Cybersecurity Framework — five plain-language functions (Govern, Identify, Protect, Detect, Respond, Recover) most companies use as a common map of "are we covering the basics."
SOC 2
An audit report proving a company's security controls actually work, not just exist on paper. The one enterprise customers ask for before they'll sign a contract.
ISO 27001
An international certification for having a formal Information Security Management System (ISMS) — heavier and more globally recognized than SOC 2, common outside the US.
GDPR
EU law governing how personal data is collected, stored, and used. Applies if you have any EU users, regardless of where your company is based.
CCPA
California's version of GDPR — gives California residents rights over their personal data.
HIPAA
US law protecting health data. Relevant if you touch anything medical, even indirectly (e.g. a health-tech vendor).
PCI-DSS
The standard for handling credit card data. Applies if you process, store, or transmit card payments.
RMF
NIST's Risk Management Framework — a formal, step-by-step process (mostly used by government and contractors) for authorizing a system as secure enough to operate.
Core Concepts
CIA Triad
Confidentiality, Integrity, Availability — the three things every security control is ultimately protecting.
MFA
Multi-Factor Authentication — requiring more than a password to log in (e.g. password + phone code). The single highest-impact control most companies can turn on immediately.
Zero Trust
A security model that assumes no user or device is automatically trusted, even inside the company network — every request gets verified.
CVE
Common Vulnerabilities and Exposures — a public ID number given to a specific known security flaw (e.g. "CVE-2024-1234"), so everyone can reference the same bug.
EDR
Endpoint Detection and Response — software installed on laptops/servers that watches for and can automatically respond to suspicious activity.
SIEM
Security Information and Event Management — a system that collects logs from everywhere (servers, apps, network gear) into one place so analysts can spot patterns.
IAM
Identity and Access Management — the systems and policies controlling who can access what.
DFIR
Digital Forensics and Incident Response — the discipline of figuring out what happened after a breach and preserving evidence properly.
GRC
Governance, Risk, and Compliance — the function that translates security work into policy, risk register entries, and audit-ready documentation.
C2
Command and Control — the infrastructure an attacker (or a red teamer simulating one) uses to remotely control a compromised machine.
Certifications
ISC2 CC
Certified in Cybersecurity — ISC2's free, entry-level cert. Good first credential for anyone non-technical.
Security+
CompTIA's baseline cert, widely recognized as the industry-standard starting credential.
CISSP
Certified Information Systems Security Professional — the broad, senior-level cert most CISOs hold. Requires years of experience to actually sit the exam.
CISM / CISA / CRISC / CGEIT
ISACA's suite of certs — CISM (security management), CISA (auditing), CRISC (risk), CGEIT (IT governance). Aimed at people who manage programs, not configure tools.
OSCP
Offensive Security Certified Professional — the widely respected, fully hands-on penetration testing certification. You have to actually hack a live environment to pass.
PEH / PJPT / PNPT
TCM Security's practical pentesting ladder — Practical Ethical Hacking (course), Practical Junior Penetration Tester, Practical Network Penetration Tester (certs), each fully hands-on.
GCFA / GCFE / GNFA
SANS/GIAC forensics certifications — GCFE (examiner, more foundational), GCFA (forensic analyst), GNFA (network forensics), each progressively deeper.
CCNA / CCNP
Cisco's networking certification ladder — CCNA is the standard entry cert, CCNP is the advanced/specialist tier (e.g. CCNP Security).
CCSK / CCSP
Cloud security certs — CCSK (Certificate of Cloud Security Knowledge, vendor-neutral, more foundational) and CCSP (Certified Cloud Security Professional, ISC2's advanced cert).
Tools
Nmap
Free tool for scanning a network to see what devices and services are exposed — the standard first step in almost any security assessment.
Wireshark
Free tool for capturing and inspecting network traffic packet-by-packet.
Metasploit
A framework of pre-built exploits used by penetration testers to test whether known vulnerabilities are actually exploitable.
Burp Suite
The standard tool for testing web application security — intercepts and lets you manipulate traffic between browser and server.
Splunk
A widely used SIEM platform for searching and visualizing log data at scale.
Volatility
A free tool for analyzing a computer's memory (RAM) after an incident, to see what was running at the time of compromise.
Terraform
A tool for defining cloud infrastructure as code — relevant to security because misconfigurations can be caught before they're ever deployed.
